# STM32 Mill 第一阶段收口结论

本文档用于对第一阶段“远程参数维护”进行阶段性收口总结。

## 1. 第一阶段范围

第一阶段已完成的目标：

- 远程读取现场数据
- 远程读取配置区
- 远程读取维护状态区
- 远程读取诊断区
- 远程解锁
- 远程写配置寄存器
- 远程提交保存到 Flash
- 远程恢复默认参数
- 远程参数校验与错误码细分
- 最近非法写入地址和值记录
- 最近非法读取地址记录
- 最近维护命令结果记录
- 自动/手动模式切换框架

第一阶段明确不包含：

- 具体业务自动控制逻辑
- Bootloader
- 本地升级
- 远程升级
- OTA
- 云平台页面化配置流程

## 2. 已完成实现

当前第一阶段已具备以下能力：

- 数据寄存器区、配置寄存器区、维护寄存器区、诊断寄存器区
- 参数暂存、提交保存、恢复默认、清错误
- Flash 持久化与上电重载
- 参数边界检查
- 参数组合关系检查
- 远程维护状态与错误码回读
- 版本号、编译时间、运行时长、上电次数、重启原因等诊断信息
- Modbus CRC 错误计数与 UART 异常计数

## 3. 本轮实测结果

本轮已完成并通过的验收项：

- 新固件识别正常
- 现场数据读取正常
- 配置区读取正常
- 合法参数修改正常
- 非法参数拦截正常
- 错误口令拦截正常
- 非法维护命令记录正常
- 非法读取记录正常
- 诊断寄存器整体工作正常

本轮实测已确认的关键结果包括：

- 固件版本与协议版本可远程读取
- 编译日期时间可远程读取
- 配置寄存器可远程修改并提交保存
- 非法参数写入会返回异常响应并记录地址和值
- 错误解锁口令会返回异常响应并记录地址和值
- 非法维护命令会记录到最近一次维护命令结果
- 越界读取会记录最近一次非法读取起始地址

## 4. 当前收口判断

结论：

- 第一阶段“远程参数维护框架”已经达到阶段性收口条件
- 现阶段可以停止继续扩第一阶段功能
- 后续工作建议转入第二阶段方案设计

## 5. 当前遗留项

建议补做但不影响当前收口判断的事项：

- 做一次“严格意义上的掉电/重启保持验收记录”

说明：

- 该项属于验收闭环补齐
- 不属于新增功能开发
- 即使后续会建设日志系统，这一步仍建议保留

## 6. 为什么仍建议做掉电/重启保持验收

原因如下：

- 日志系统是后续能力，当前还未落地，不能代替本阶段验收
- 参数是否真正保存在 Flash，是第一阶段最核心的交付能力之一
- 现场问题里，最常见的风险之一就是“改完参数看似生效，但重启后丢失”
- 这一项验证成本低，但能明确排除一类高风险问题

因此建议：

- 仍然补做一次掉电/重启保持测试
- 但它可以作为“收口补记录项”，不必阻塞第二阶段方案准备

## 8. 阶段性结论

第一阶段到当前为止，可以认定为：

- 已完成
- 已实测通过主要功能
- 可作为后续第二阶段的稳定基础
