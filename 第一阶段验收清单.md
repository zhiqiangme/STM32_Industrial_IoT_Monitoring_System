# STM32 Mill 第一阶段验收清单

本文档用于对第一阶段“远程参数维护”做阶段性收口验收。

第一阶段验收目标：

- 远程读取现场数据正常
- 远程读取配置/维护/诊断寄存器正常
- 远程修改参数正常
- 参数保存到 Flash 并在重启后保留
- 非法写入/非法读取可被拦截并记录
- 诊断寄存器可用于远程排障

当前阶段不包含：

- 业务自动控制逻辑
- 本地升级 / 远程升级
- OTA / Bootloader
- 云平台页面化配置

## 1. 测试准备

测试前确认：

- STM32 已烧录当前第一阶段固件
- G780S 已联网并可透传 Modbus RTU
- 有人云网络调试已打开 `HEX` 模式
- 每次发一条命令，等待返回后再发下一条

## 2. 验收项目

### 2.1 新固件识别

发送：

```text
0a030038001384b1
```

预期：

- 能正常返回一帧 `0a03...`
- `0x0038 = 0x0100`
- `0x0039 = 0x0100`
- `0x003A~0x003C` 编译时间非全 0

判定：

- 满足以上条件则通过

### 2.2 现场数据读取

发送：

```text
0a04000400017170
```

预期：

- 能正常返回温度寄存器数据

判定：

- 返回有效数据则通过

### 2.3 配置区读取

发送：

```text
0a0300200009857d
```

预期：

- 能正常返回 `0x0020 ~ 0x0028`
- 可读到采集周期、流量采样周期、DI 去抖、温差阈值、流量参数、控制模式

判定：

- 配置区可完整读取则通过

### 2.4 合法参数修改

步骤：

1. 解锁

```text
0a060030a55a73d5
```

2. 将采集周期改为 `10000ms`

```text
0a06002027109347
```

3. 提交保存

```text
0a060031000118be
```

4. 读取状态区

```text
0a0300320006657c
```

5. 读取配置区

```text
0a0300200009857d
```

预期：

- 解锁成功并回显
- 写入成功并回显
- 提交保存成功并回显
- `0x0033 = 0`
- `0x0035~0x0036` 配置序号递增
- `0x0020 = 0x2710`

判定：

- 满足以上条件则通过

### 2.5 参数掉电/重启保持

步骤：

1. 完成一次合法参数修改并提交保存
2. 让设备重启或重新上电
3. 重新读取配置区

发送：

```text
0a0300200009857d
```

预期：

- `0x0020` 仍为之前保存值
- 诊断区 `0x003F~0x0040` 上电次数增加

判定：

- 参数保留且上电次数增加则通过

### 2.6 非法参数拦截

步骤：

1. 解锁

```text
0a060030a55a73d5
```

2. 故意写入非法采集周期 `100ms`

```text
0a06002000648890
```

3. 读取状态区

```text
0a0300320006657c
```

4. 读取诊断区

```text
0a030038001384b1
```

预期：

- 写非法值时返回异常帧 `0a860373a3`
- `0x0033 = 0x0009`
- `0x0042 = 0x0020`
- `0x0043 = 0x0064`

判定：

- 非法值被拒绝且地址/数值被记录则通过

### 2.7 错误口令拦截

步骤：

1. 故意发送错误解锁口令

```text
0a060030123485c9
```

2. 读取状态区或诊断区

```text
0a0300320006657c
```

```text
0a030038001384b1
```

预期：

- 返回异常帧 `0a860373a3`
- `0x0033 = 0x0002`
- `0x0042 = 0x0030`
- `0x0043 = 0x1234`

判定：

- 错误口令被拒绝且地址/数值被记录则通过

### 2.8 非法维护命令记录

步骤：

1. 解锁

```text
0a060030a55a73d5
```

2. 故意发送非法维护命令

```text
0a060031ffffd8ce
```

3. 读取 `0x0049`

```text
0a030049000154a7
```

预期：

- 返回异常帧 `0a86043261`
- `0x0049 = 0x0006`

判定：

- 最近一次维护命令结果被正确记录则通过

### 2.9 非法读取记录

步骤：

1. 故意读取越界地址

```text
0a03004a001064ab
```

2. 再读取 `0x004A`

```text
0a03004a0001a4a7
```

预期：

- 越界读取返回异常帧 `0a8302b133`
- `0x004A = 0x004A`

判定：

- 非法读取起始地址被正确记录则通过

### 2.10 诊断信息完整性

发送：

```text
0a030038001384b1
```

预期：

- `0x0038~0x0039` 固件/协议版本正常
- `0x003A~0x003C` 编译日期时间正常
- `0x003D~0x003E` 运行时长递增
- `0x003F~0x0040` 上电次数合理
- `0x0041` 重启原因合理
- `0x0044` 配置来源合理
- `0x0045~0x0046` CRC 错误计数可累计
- `0x0047~0x0048` UART 异常计数可读
- `0x0049` 最近维护命令结果可读
- `0x004A` 最近非法读取地址可读

判定：

- 诊断寄存器均可读且数值符合预期则通过
