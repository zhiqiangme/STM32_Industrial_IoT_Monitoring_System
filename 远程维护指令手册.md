# STM32 Mill 远程维护指令手册

本文档用于说明当前第一阶段“远程参数维护”功能的寄存器定义、操作流程和常用 HEX 报文。

当前远程维护链路为：

`远端调试工具/服务器 -> G780S 网络透传 -> USART3 -> STM32 Modbus RTU 从机(地址 10)`

注意：

- 当前使用的是 `Modbus RTU` 原始帧，不是网络 AT 指令
- 发送时必须使用 `HEX` 模式
- 每条指令都必须包含正确的 `CRC16`
- 写配置区只是暂存，必须执行“提交保存”才会写入 Flash 并生效

## 1. 通信参数

- 从站地址：`0x0A`，十进制 `10`
- 协议：`Modbus RTU`
- 当前物理通道：`USART3`
- G780S 作用：网络透传，不负责解释 STM32 参数含义

## 3. 远程配置寄存器区

这部分寄存器用于远程维护。

重要说明：

- 写入前必须先解锁
- 写入后只是暂存
- 必须执行“提交保存”才会写入 Flash

| 地址 | 名称 | 类型 | 单位/格式 | 默认值 | 说明 |
|---|---|---|---|---|---|
| `0x0020` | `REG_CFG_SENSOR_PERIOD_MS` | R/W | `uint16, ms` | `2000` | 现场传感器采集周期 |
| `0x0021` | `REG_CFG_FLOW_SAMPLE_MS` | R/W | `uint16, ms` | `1000` | 流量换算采样周期 |
| `0x0022` | `REG_CFG_DI_DEBOUNCE_MS` | R/W | `uint16, ms` | `50` | DI 去抖时间 |
| `0x0023` | `REG_CFG_TEMP_THRESHOLD_X10` | R/W | `uint16, ×10℃` | `10` | 温度变化触发阈值 |
| `0x0024` | `REG_CFG_PPL_X100_H` | R/W | `uint32 高 16 位` | `0x0001` | 每升脉冲数高 16 位 |
| `0x0025` | `REG_CFG_PPL_X100_L` | R/W | `uint32 低 16 位` | `0x01D0` | 每升脉冲数低 16 位 |
| `0x0026` | `REG_CFG_HZ_PER_LPM_X100_H` | R/W | `uint32 高 16 位` | `0x0000` | Hz/LPM 高 16 位 |
| `0x0027` | `REG_CFG_HZ_PER_LPM_X100_L` | R/W | `uint32 低 16 位` | `0x044C` | Hz/LPM 低 16 位 |
| `0x0028` | `REG_CFG_CONTROL_MODE` | R/W | `uint16` | `0` | 控制模式，0=手动，1=自动 |

### 3.1 当前配置寄存器实际含义

#### `0x0020 REG_CFG_SENSOR_PERIOD_MS`

- 含义：STM32 主循环里进行一轮现场采集的周期
- 影响：
  - 温度读取
  - 称重读取
  - 继电器状态读取
  - 数据刷新到 G780S 寄存器镜像

#### `0x0021 REG_CFG_FLOW_SAMPLE_MS`

- 含义：流量计换算用的采样周期
- 影响：`Flowmeter_Update()` 的计算节拍

#### `0x0022 REG_CFG_DI_DEBOUNCE_MS`

- 含义：继电器输入 DI 的去抖时间

#### `0x0023 REG_CFG_TEMP_THRESHOLD_X10`

- 含义：温度变化达到该阈值后，`PUSH_SEQ` 才递增
- 单位：`0.1℃`
- 例如：
  - 写 `10` 表示 `1.0℃`
  - 写 `25` 表示 `2.5℃`

#### `0x0024~0x0025 REG_CFG_PPL_X100`

- 含义：每升脉冲数
- 存储方式：放大 100 倍的 32 位整数
- 当前默认值：
  - `660.00`
  - 十进制存储为 `66000`
  - 十六进制为 `0x000101D0`

#### `0x0026~0x0027 REG_CFG_HZ_PER_LPM_X100`

- 含义：频率换算到 L/min 的系数
- 存储方式：放大 100 倍的 32 位整数
- 当前默认值：
  - `11.00`
  - 十进制存储为 `1100`
  - 十六进制为 `0x0000044C`

#### `0x0028 REG_CFG_CONTROL_MODE`

- 含义：控制模式
- 取值：
  - `0` = 手动模式
  - `1` = 自动模式

当前实现行为：

- 手动模式：
  - 允许本地按键控制继电器
  - 允许远程 `REG_RELAY_CTRL` 和 `REG_RELAY_CMD_BITS` 控制继电器
- 自动模式：
  - 当前暂时屏蔽本地按键和远程直接继电器控制
  - 仍保留现场采集、状态上报和参数维护能力
  - 后续自动控制逻辑将在此模式下接入

### 5.1 重启原因定义

| 值 | 含义 |
|---|---|
| `0` | 未知 |
| `1` | 上电复位 |
| `2` | NRST 引脚复位 |
| `3` | 软件复位 |
| `4` | 独立看门狗复位 |

## 8. 状态位定义

读取 `0x0032 REG_MAINT_STATUS`。

| 位 | 宏名 | 含义 |
|---|---|---|
| bit0 | `G780S_STATUS_UNLOCKED` | 当前处于解锁状态 |
| bit1 | `G780S_STATUS_STAGED_DIRTY` | 配置区已修改但尚未提交 |
| bit2 | `G780S_STATUS_CONFIG_LOADED` | 当前配置来自 Flash 成功加载 |
| bit3 | `G780S_STATUS_SAVE_OK` | 最近一次保存成功 |
| bit4 | `G780S_STATUS_ERROR` | 当前存在错误 |

例如：

- `0x000D = 0000 0000 0000 1101b`
- 表示：
  - 已解锁
  - 配置来自 Flash
  - 最近一次保存成功

## 9. 错误码定义

读取 `0x0033 REG_MAINT_LAST_ERROR`。

| 错误码 | 宏名 | 含义 |
|---|---|---|
| `0` | `G780S_ERR_NONE` | 无错误 |
| `1` | `G780S_ERR_LOCKED` | 未解锁，拒绝写入 |
| `2` | `G780S_ERR_INVALID_UNLOCK_KEY` | 解锁口令错误 |
| `3` | `G780S_ERR_FLASH_ERASE` | Flash 擦除失败 |
| `4` | `G780S_ERR_FLASH_PROGRAM` | Flash 编程失败 |
| `5` | `G780S_ERR_FLASH_VERIFY` | Flash 回读校验失败 |
| `6` | `G780S_ERR_BAD_COMMAND` | 命令非法 |
| `7` | `G780S_ERR_FLASH_EMPTY` | Flash 参数页为空，已回退默认值 |
| `8` | `G780S_ERR_FLASH_CRC` | Flash 参数页 CRC/版本错误，已回退默认值 |
| `9` | `G780S_ERR_SENSOR_PERIOD_RANGE` | 现场传感器采集周期越界 |
| `10` | `G780S_ERR_FLOW_SAMPLE_RANGE` | 流量采样周期越界 |
| `11` | `G780S_ERR_DI_DEBOUNCE_RANGE` | DI 去抖时间越界 |
| `12` | `G780S_ERR_TEMP_THRESHOLD_RANGE` | 温差触发阈值越界 |
| `13` | `G780S_ERR_PPL_RANGE` | 每升脉冲数越界 |
| `14` | `G780S_ERR_HZ_PER_LPM_RANGE` | Hz/LPM 换算参数越界 |
| `15` | `G780S_ERR_CONTROL_MODE_INVALID` | 控制模式非法 |
| `16` | `G780S_ERR_CONFIG_CONFLICT` | 参数组合关系不合法 |

### 8.1 当前已启用的校验规则

- `REG_CFG_SENSOR_PERIOD_MS`：`200~60000ms`
- `REG_CFG_FLOW_SAMPLE_MS`：`100~10000ms`
- `REG_CFG_DI_DEBOUNCE_MS`：`10~5000ms`
- `REG_CFG_TEMP_THRESHOLD_X10`：`1~500`
- `REG_CFG_PPL_X100`：`100~10000000`
- `REG_CFG_HZ_PER_LPM_X100`：`10~1000000`
- `REG_CFG_CONTROL_MODE`：只能写 `0` 或 `1`
- 整体组合关系：
  - `flow_sample_period_ms <= sensor_period_ms`
  - `di_debounce_ms < sensor_period_ms`

说明：

- 16 位参数在单寄存器写入时就会直接做范围拦截
- 32 位参数因为拆成高低寄存器写入，最终范围在“提交保存”时统一校验

## 10. 常用读取报文

### 10.1 读取维护状态区 `0x0032` 开始 6 个寄存器

发送：

```text
0a0300320006657c
```

示例返回（示意，末尾 CRC 以设备实际返回为准）：

```text
0a030c000d0000000100000002000992dd
```

解析：

- `0x0032 = 0x000D` 状态位
- `0x0033 = 0x0000` 最近错误码
- `0x0034 = 0x0001` 配置结构版本
- `0x0035~0x0036 = 0x00000002` 当前配置序号
- `0x0037 = 0x0009` 解锁剩余 9 秒

### 10.2 读取配置区 `0x0020` 开始 9 个寄存器

发送：

```text
0a0300200009857d
```

示例返回：

```text
0a0312271003e80032000a000101d00000044c0000xxxx
```

解析：

- `0x0020 = 0x2710 = 10000ms`
- `0x0021 = 0x03E8 = 1000ms`
- `0x0022 = 0x0032 = 50ms`
- `0x0023 = 0x000A = 1.0℃`
- `0x0024~0x0025 = 0x000101D0 = 66000 = 660.00`
- `0x0026~0x0027 = 0x0000044C = 1100 = 11.00`
- `0x0028 = 0x0000 = MANUAL`

### 10.3 读取诊断区 `0x0038` 开始 17 个寄存器

发送：

```text
0a03003800110570
```

返回格式说明：

- `0x0038~0x0039`：固件版本、协议版本
- `0x003A~0x003C`：编译日期时间
- `0x003D~0x003E`：运行时长秒数
- `0x003F~0x0040`：上电次数
- `0x0041`：重启原因
- `0x0042~0x0043`：最近非法写入地址和值
- `0x0044`：当前配置来源
- `0x0045~0x0046`：Modbus CRC 错误次数
- `0x0047~0x0048`：UART 异常次数

## 13. 使用注意事项

- 不要把两条 RTU 帧直接拼成一条连续发送
- 每条指令发完，等返回后再发下一条
- 改了寄存器值以后，不执行“提交保存”不会落到 Flash
- 解锁超时以后，设备会自动重新上锁
- 如果写寄存器返回异常帧：
  - `86 02`：通常表示地址非法
  - `83 02`：通常表示读取地址非法
- 读写功能正常时，回包一般会直接回显原命令或返回 `0x03` 数据帧

## 14. 当前阶段边界

当前第一阶段已经实现的是“远程参数维护框架”，不是“业务逻辑参数系统”。

已经支持远程修改的是：

- 采集周期
- 流量采样周期
- DI 去抖
- 温差触发阈值
- 流量换算参数
- 自动/手动模式切换

当前还没有实现的是：

- 重量达到阈值后自动打开继电器
- 温度达到阈值后自动打开继电器
- 业务规则联锁逻辑

这些要等后续你把现场逻辑确定后，再扩展新的业务参数寄存器。
